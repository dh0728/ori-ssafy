# 주제 : P2P 지인 소액 대출 플랫폼

## 🏷️ 개발 배경:

지인 간 돈을 빌려주고 돌려받는 과정에서 기한 초과, 연체, 불법 채무 독촉 등의 문제가 발생할 수 있음.
소액 대출의 경우 법적 절차 진행이 어려워 채권자가 손해를 보는 경우가 많음.
채무 독촉 시 법적 제재 사항(일정 시간 이후 독촉 금지, 주변인에게 알리는 행위 금지 등)을 모르고 위반하는 사례가 존재함.
이러한 문제를 해결하기 위해 지인 간의 돈 거래 시 거래 내역과 대출 증거를 관리하고 중계할 수 있는 앱을 개발하고자 함.

차용증 작성과 채권 관리에 도움을 주는 서비스

### 📌 기능

💰 차용증 작성

- 1분만에 작성 가능한 전자 차용증 서비스 제공

기술 스택
- iText
- Apache PDFBox
- JasperReports

💰 1 : 1 채팅기능

- 채팅을 통해 채무 관련 대화를 함으로써 기록하여 증빙 자료로 보관
- 법적 문제 발생 가능성 감지: AI를 이용해 불법 채무 독촉(협박, 명예훼손, 불법 상환 요구 등) 감지 후 경고 알림 제공.
- 필요 시 채팅 기록을 PDF로 저장하여 법적 증거로 활용 가능.

기술스택

- WebSocket (STOMP) 기반
    ✅ 장점
    - **실시간 채팅**에 적합 (지연 시간이 거의 없음 🚀)
    - **WebSocket 연결 유지**를 통해 빠른 메시지 송수신
    - **단순한 메시지 흐름** → 메시지 생성 → 브로드캐스트 → 즉시 반응
    - **구독(Pub/Sub) 기반 채널 운영 가능** → 특정 채팅방에만 메시지 전달
    
    ❌ 단점
    - **WebSocket 연결 유지 필요** → 연결이 끊기면 메시지를 받을 수 없음
    - **메시지 내구성이 없음 (Persistence X)** → 서버가 다운되면 메시지 유실 가능
    - **대규모 트래픽을 처리하기 어렵다** → 많은 사용자가 참여하면 부하가 증가
- Kafka
    ✅장점 
    - **메시지 내구성 보장 (Persistence O)** → 메시지를 Kafka에 저장하므로 서버가 다운되더라도 복구 가능
    - **수많은 사용자도 확장 가능** → Kafka의 분산 처리 시스템을 활용
    - **대량의 메시지를 처리할 수 있음** → WebSocket보다 대량 트래픽 처리에 강함
    - **비동기 처리 가능** → 메시지를 저장하고 후처리 가능 (예: AI 분석, 로그 저장)
    ❌ 단점 
    - **실시간성이 다소 부족** → Kafka는 기본적으로 메시지를 저장하고 가져오는 방식이므로, WebSocket보다는 약간의 지연이 발생
    - **구현이 복잡함** → Kafka를 구축하고 메시지를 처리하는 시스템을 추가해야 함
    - **1:1 채팅에는 과할 수 있음** → 작은 규모의 채팅 시스템에서는 불필요하게 무거울 수 있음

구현 방법
1. Spring Boot + WebSocket (STOMP) + Redis
    📌 언제 적합한가?
    - 실시간 메시지 전송이 최우선
    - 서버 부하가 많지 않은 소규모 시스템
    - 메시지 저장보다는 빠른 전달이 중요
    - 일시적 메시지 처리가 가능하며, 메시지 유실이 문제가 되지 않는 경우

2. Spring Boot + Kafka + WebSocket(MQTT) 하이브리드
    1. WebSocket & MQTT 기반 실시간 메시징
        - MQTT (Message Queuing Telemetry Transport) 프로토콜을 이용해 메시지를 최소한의 네트워크 비용으로 전송
        - WebSocket (STOMP) 기반으로 연결 유지 → 빠른 실시간 전송 지원
        - STOMP가 일반적인 메시징 프로토콜이라면, MQTT는 저전력/저대역폭 환경에서도 효율적으로 작동하도록 최적화된 프로토콜
        
        ✅ 왜 WebSocket이 아니라 MQTT를 쓸까?
        - WebSocket은 빠르지만, 모바일 환경에서 연결 유지 비용이 크다
        - MQTT는 초저전력(배터리 소모 적음), 네트워크 상태가 좋지 않아도 유지 가능
        - 모바일 기기가 네트워크가 끊겼다가 다시 연결될 때, MQTT는 자동으로 재연결하여 데이터 손실 방지
    
    2. Kafka(또는 자체 메시징 시스템) 기반 메시지 큐
        - Kafka를 활용한 분산 메시지 브로커를 운영하여 수억 개 이상의 메시지를 처리
        - Kafka가 아니라 자체 메시징 시스템을 운영할 가능성도 있음 (초고속 전송을 위해 커스텀 최적화)
        - 메시지를 대량의 서버에 분산 저장하고, 필요할 때 빠르게 가져올 수 있도록 설계

    3. NoSQL (HBase / RocksDB) 기반 메시지 저장
        - 기존 RDBMS(MySQL, PostgreSQL)는 수백만 건의 메시지를 실시간으로 처리하는 데 비효율적
        - 카카오톡은 HBase(컬럼 기반 NoSQL) 또는 RocksDB(Key-Value Store) 같은 고속 읽기/쓰기가 가능한 DB를 사용
        - 메시지는 Kafka에서 HBase로 전송되며, 사용자 요청이 있을 때 불러옴
        
        ✅ 왜 HBase, RocksDB 같은 NoSQL을 사용할까?
        - 쓰기 성능이 뛰어남 → 초당 수백만 개의 메시지를 저장 가능
        - 수평 확장이 쉬움 → 채팅방이 많아져도 서버를 쉽게 추가할 수 있음
        - 컬럼 기반 구조라 메시지 검색이 빠름

     4. FCM / APNS 기반 푸시 시스템
        - 상대방이 오프라인 상태면 푸시 메시지를 Firebase Cloud Messaging(FCM) 또는 Apple Push Notification Service(APNS)를 통해 전송
        - 읽음/안 읽음 상태 관리 → 메시지 수신 후 Kafka 또는 DB에서 상태 업데이트
        ✅ 푸시 메시지 최적화
        - 오프라인 유저에게는 즉시 푸시 알림
        - 온라인 유저에게는 WebSocket을 통해 직접 전달

    📌 언제 적합한가?
    - 대규모 트래픽을 처리해야 하는 경우
    - 메시지 저장이 필요하고 유실되면 안 되는 경우
    - 확장성이 중요한 경우 (대량의 사용자가 접속하는 플랫폼)
    - 실시간 채팅 + AI 분석 + 메시지 아카이빙까지 고려할 때

💰 이체 및 거래내역 관리 기능

- 앱 내 송금 기능 제공하여 대출/상환 기록을 자동으로 저장.
- 은행 계좌와 연동하여 자동으로 거래 내역을 기록하고 채권자가 손쉽게 확인할 수 있도록 제공.
- 이체 내역을 통해 언제, 얼마를 빌려주고 상환했는지 증빙 가능.

💰 대출 계약 및 약정 관리

- 대출 전 자동 계약서 생성 기능 제공 (기본 템플릿 제공).
- 전자 서명 또는 **간편 인증(예: 카카오 인증, 본인 인증)**을 통한 계약 동의 가능.
- 이자율, 상환 방식(일시불/분할), 기한 설정 후 저장 가능.
- 계약 내용 위반 시 자동 경고 알림 제공.

기술 스택

💰 상환 리마인더 및 알림 시스템

- 상환 기한 전 자동 알림 발송.
- 채무자가 미납할 경우 자동 상환 요청 메시지 전송.
- 연체 시 페널티 설정 가능 (채권자가 사전 설정한 조건에 따라 추가 이자 또는 연체료 발생).
- 채무자가 상환 일정을 조정할 경우, 서로 합의 후 일정 변경 가능.

💰 신용평가 및 신뢰도 시스템

- 신뢰도 점수 시스템 제공: 상환 이력을 기반으로 채무자의 신뢰도를 평가 (예: 100점 만점 기준, 연체 여부 반영).
- 대출 및 상환 완료 후 서로 피드백(별점, 리뷰) 가능.
- 만약 다른 지인과의 거래시 신뢰도 점수를 통해 신뢰 여부를 확인(당근 온도)

    - "누가 가장 좋은 조건으로 돈을 빌려줄 것인가?"

    - 이 과정에서 짧은 시간 동안 대량의 요청이 발생
    - 특정 시간 안에 가장 높은 이자율을 제시한 사람에게 대출이 성사됨

    - 특정 시간에 한정된 인원만 초저금리 대출 가능
    - 선착순으로 대출을 신청해야 하기 때문에 트래픽이 순간적으로 폭증
    - 콘서트 예매처럼 수천 명이 동시에 대출을 신청하는 상황 발생 가능

# 주제2 재능있는 아이들 후원 플랫폼

## 개발 배경:
- 재능이 있으나 경제적으로 어려움있어 꿈을 펼치지 못하고 있는 아이들이 많이 있다.
- 재능 있는 아이들이 자신의 꿈을 실현할 수 있도록 돕도록 하는 플랫폼이 필요하다
- 재능 있는 아이들이 경제적 어려움 없이 꿈을 펼칠 수 있도록 투명하고 지속 가능한 후원 환경을 조성하기 위한 플랫폼이 있으면 좋지 않을까 하는 생각에서 시작
- 아이들은 자신의 스토리나 재능을 보여줄 수 있는 영상이나 글을 업로드하여 후원을 유도할 수 있음.
- 후원자들은 아이들의 스토리와 재능을 확인한 후, 원하는 아이에게 직접 후원할 수 있다.

## 기능 
1. 실시간 라이브 방송 및 후원 시스템
- 실시간 스트리밍 기능 (아이들이 후원자와 직접 소통 가능)
- 실시간 후원 (Super Chat 또는 후원 메시지 포함)
기술스택
 - WebRTC


2. 후원금 관리 및 투명한 사용 내역 제공
- 후원자가 후원 내역 확인 (누구에게 후원했는지, 금액)
- 후원금 사용 내역 등록 및 증빙 자료 업로드
- 후원자들에게 사용 내역 자동 보고

3. 정기 후원 및 자동 결제 시스템
- 후원자가 특정 아이에게 매월 정기 후원 가능
- 신용카드, 계좌이체, 가상자산 결제 지원
- 후원 일정 변경 및 취소 기능

4. 후원 트랜잭션 모니터링
- 후원 요청 심사 및 승인 기능
- 비정상적인 후원 감지 (예: 자금 세탁 방지)
- 사용자 및 후원 데이터 모니터링

5. 실시간 후원금액랭킹 만들기
- 후원 순위 실시간 업데이트 → 후원 금액에 따라 즉시 랭킹 반영

6. 커뮤니티 & 멘토링 시스템
- 후원자들이 아이들과 지속적인 교류 가능 (게시판, 댓글 기능, 1대1 채팅)
- 전문가 또는 재능 있는 후원자가 멘토링 요청 가능
- 후원자가 자신의 경험을 공유할 수 있는 스토리 게시판

7. AI 기반 후원 추천 시스템 (이건 다 하기 싫을 듯 하지만 일단 넣기)
- 후원자의 관심사 및 후원 내역을 분석하여 추천 아이를 제공
- 아이들의 재능·스토리 기반 태그를 활용한 개인화된 추천
- RAG 기반으로 후원자가 원하는 조건(예: 음악, 미술, 스포츠 등)에 맞는 아이들 검색
